Kiwi.Plugins.DamagePipeline.MeterNode Class

Extends Kiwi.Plugins.DamagePipeline.PipelineNode
Defined in: src/pipeline-node-meter.js:7
Module: MeterNode
Parent Module: Kiwi.Plugins

Available since 0.1.0

Meter Node, a pipeline object that extends the standard Pipeline Node with a meter.

The Meter Node has several built-in functions. Each receives a Pack as its argument. You can override these to perform custom functionality.

  • doOnReceive: called when the node receives a Pack. Default behaviour: Apply Pack to meter
  • doOnZero: called when the node drops to 0. Default behaviour: None
  • doOnMax: called when the node rises to maximum. Default behaviour: None
  • doOnBreak: called when the node drops below 0. Default behaviour: None
  • doOnOverflow: called when the node rises above maximum. Default behaviour: None


The Pipeline Node inherits all functionality of the Pipeline Node, but it implements a custom version of the _operation method. You should override the above functions instead. As an extension of PipelineNode, the MeterNode will automatically process all packs and subpacks in bottom-up order. You do not need to write methods to deal with subpacks.

doOnReceive

(
  • pack
)
public

Defined in src/pipeline-node-meter.js:246

Available since 0.1.0

Default method executed upon receiving a Pack. This will transfer value from the Pack to the meter's value, according to the pack mode (ADD, SUBTRACT, or SET). If there is any value left over, it will remain in the Pack. It is not recommended that you override this function.

When receiving a Pack with mode SET, this function will assign as much value to this meter as it can fit. Any leftover value will remain in the Pack. This is intended to trigger overflow behaviour. It might create unexpected results if the pack is not discarded.

process

() public

Inherited from Kiwi.Plugins.DamagePipeline.PipelineNode: src/pipeline-node-base.js:417

Available since 0.1.0

Performs the node operation on all current packs. The operation is only performed on those packs which match a tag of this node. If this node has no tags, all nodes are processed. All nodes are subsequently sent to dispatchDefault unless doDefaultDispatch has been set to false.

The process goes through each pack bottom-up; the deepest children are evaluated first, working up to the root pack.

If you want to evaluate top-down, set processTopDown to true. Be careful when evaulating top-down, as you may cause packs to be discarded with a parent before they are processed.

This is normally called directly after damage is received. However, if you have not set the processOnReceive flag, it will not execute immediately. You can call it according to your own requirements.

processTopDown

Boolean public

Inherited from Kiwi.Plugins.DamagePipeline.PipelineNode: src/pipeline-node-base.js:376

Available since 0.1.0

Whether to process a pack top-down; that is, the master pack followed by its children. This is DISABLED by default. If a top-level pack is emptied, its children will be discarded before they can be processed. For this reason we normally process bottom-up, doing the children first then their parent. However, in some cases it may be useful to extract the topmost case of a tag, not the bottommost, so this flag is provided.

Default: false
